Main functional flow (current + next)
1) MT5 gateway initializes
2) Market data service loads bars per symbol
3) Feature engine builds locked-v1 feature snapshot
4) Candidate engine detects BUY/SELL candidate (or NO_CANDIDATE)
5) State guard blocks invalid runtime states (open position / one trade per bar / cooldown)
6) Logger engine writes structured logs (snapshot / candidate / guard)
7) Symbol registry resolves alias -> canonical symbol (analysis symbol)
8) Decision validator blocks duplicate underlying setup across aliases
9) Candidate pipeline postprocessor produces runtime-safe report + logs
10) Next: AI decision confirms/denies candidate
11) Next: execution engine sends order to broker-specific execution symbol
